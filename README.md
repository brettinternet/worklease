# worklease

`worklease` is a provider-neutral Python distribution for same-host work-lease coordination. The package currently exposes the stable bootstrap CLI; lease storage and provider adapters are added by subsequent implementation tasks.

## Requirements

- Python 3.14 or newer
- `uv` (the repository uses the latest version through mise)
- `pyright` for type checking
- POSIX host support is the target for the coordination implementation

## Development

The repository pins its development workflow through `mise.toml`:

```sh
mise exec uv -- uv lock
mise exec uv -- uv sync --locked
mise run sync
mise run test
mise run typecheck
mise run build
```

`uv.lock` is generated by `uv lock` and committed. `uv sync --locked` refuses to change it, so a clean checkout reproduces the locked environment.

Run the editable local console script without activating the virtual environment:

```sh
mise exec uv -- uv run worklease --version
mise exec uv -- uv run worklease --format text --version
```

Install the current checkout as an editable uv tool when a shell-wide command is useful:

```sh
mise exec uv -- uv tool install --editable .
worklease --version
```

The release installer is exposed as `mise run install-release VERSION=vX.Y.Z` once a tagged release is available. Local release installation requires `WORKLEASE_REPOSITORY=owner/name` so the repository identity is explicit.

## CLI bootstrap contract

JSON is the default output format. Version responses contain `schemaVersion`, `operation`, `ok`, and the packaged `version`:

```json
{"schemaVersion":1,"operation":"version","ok":true,"version":"0.1.0"}
```

Use `--format text` explicitly when a bare version is required:

```text
0.1.0
```

The package version is defined once in `pyproject.toml` and exposed through the `worklease` console script.
